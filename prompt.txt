# Техническое задание

**Проект:** GUI-приложение MATLAB для ручной фазовой коррекции спектра по аналогии с TopSpin
**Назначение:** интерактивная фазировка комплексного спектра с использованием коррекции нулевого и первого порядка относительно выбранной опорной частоты (pivot / pointer)

---

## 1. Цель разработки

Разработать приложение в MATLAB с графическим интерфейсом, позволяющее:

* загружать комплексный спектр;
* визуализировать спектр;
* выбирать опорную частоту (pivot);
* выполнять ручную фазовую коррекцию:

  * нулевого порядка (`PH0`);
  * первого порядка (`PH1`);
* применять фазовую коррекцию к спектру в реальном времени;
* сохранять параметры фазировки;
* при необходимости сохранять скорректированный спектр.

Приложение должно повторять пользовательскую логику, аналогичную работе в TopSpin:

1. пользователь нажимает режим корректировки фазы;
2. выбирает точку спектра, относительно которой будет выполняться фазировка;
3. регулирует фазу нулевого порядка так, чтобы сигнал в точке pivot был корректно сфазирован;
4. регулирует фазу первого порядка для корректировки остальных сигналов;
5. сохраняет параметры фазировки.

---

## 2. Объект обработки

Приложение работает с **одномерным комплексным спектром**:

[
S[n] = Re[n] + i \cdot Im[n], \quad n = 0,1,\dots,N-1
]

где:

* `Re[n]` — действительная часть спектра,
* `Im[n]` — мнимая часть спектра,
* `N` — число точек спектра.

Дополнительно может использоваться частотная ось:

* в Гц;
* в ppm;
* либо индексная шкала, если частотная ось не задана.

---

## 3. Математическая модель

### 3.1. Фазовая коррекция

Для каждой точки спектра вычисляется фазовый множитель:

[
S_{corr}[n] = S[n]\cdot e^{i\phi[n]}
]

где общая фаза:

[
\phi[n] = \phi_0 + \phi_1 \cdot g(n, n_p)
]

где:

* `φ0` — коррекция нулевого порядка;
* `φ1` — коррекция первого порядка;
* `np` — индекс опорной точки (pivot);
* `g(n,np)` — нормированное расстояние от текущей точки до pivot.

Рекомендуемая реализация:

[
g(n,n_p)=\frac{n-n_p}{N-1}
]

Тогда:

* в точке `n = np` вклад первого порядка равен нулю;
* изменение `PH1` не изменяет фазу в pivot;
* фаза меняется линейно по спектру.

### 3.2. Единицы фазовых параметров

В GUI параметры `PH0` и `PH1` должны отображаться в **градусах**.

Для вычислений в MATLAB необходимо переводить их в радианы:

[
\phi_{rad} = \phi_{deg}\cdot \frac{\pi}{180}
]

Тогда:

[
\phi[n] = \left(PH0 + PH1 \cdot \frac{n-n_p}{N-1}\right)\cdot \frac{\pi}{180}
]

## 4. Функциональные требования

### 4.1. Загрузка данных

Программа должна поддерживать загрузку:

* MAT-файла (`.mat`);
* текстового файла (`.txt`, `.csv`) с колонками:

  * `real`, `imag`;
  * либо `freq`, `real`, `imag`.

Минимально необходимый набор данных:

* вектор действительной части;
* вектор мнимой части.

Опционально:

* вектор частотной оси.

### 4.2. Отображение спектра

GUI должно отображать:

* исходный спектр;
* скорректированный спектр;
* положение pivot.

Минимум:

* главный график действительной части скорректированного спектра;
* возможность включить отображение:

  * мнимой части;
  * исходного спектра;
  * вертикальной линии pivot.

### 4.3. Выбор pivot

Пользователь должен иметь возможность задать pivot:

* кликом мыши по графику;
* вводом индекса точки;
* вводом частоты/ppm, если доступна ось.

После выбора pivot:

* GUI должно визуально отметить выбранную точку;
* значение pivot должно отображаться в панели параметров.

### 4.4. Коррекция нулевого порядка

Пользователь должен иметь возможность изменять `PH0`:

* ползунком;
* числовым полем;
* кнопками пошагового увеличения/уменьшения.

При изменении `PH0`:

* спектр должен пересчитываться в реальном времени;
* график должен обновляться без перезапуска приложения.

### 4.5. Коррекция первого порядка

Пользователь должен иметь возможность изменять `PH1`:

* ползунком;
* числовым полем;
* кнопками пошагового увеличения/уменьшения.

При изменении `PH1`:

* спектр должен пересчитываться в реальном времени;
* фаза в pivot должна оставаться неизменной;
* график должен обновляться немедленно.

### 4.6. Сохранение параметров

После завершения фазировки пользователь должен иметь возможность сохранить:

* `PH0`;
* `PH1`;
* `pivot index`;
* `pivot frequency` или `pivot ppm`, если доступно;
* дату/время сохранения.

Сохранение должно поддерживаться в:

* MAT-файл;
* JSON или TXT-файл параметров.

### 4.7. Сохранение скорректированного спектра

Программа должна позволять сохранить:

* действительную часть скорректированного спектра;
* мнимую часть скорректированного спектра;
* частотную ось;
* параметры фазировки.

### 4.8. Сброс и отмена

Должны быть реализованы кнопки:

* **Reset** — сброс `PH0`, `PH1` и pivot к значениям по умолчанию;
* **Undo last** — откат последнего изменения параметров;
* **Apply / Save** — фиксация текущих параметров.

## 17. Псевдокод вычислительной части

```matlab
Input: complex spectrum S, ph0_deg, ph1_deg, pivotIndex
N = length(S)

for n = 1:N
    g = (n - pivotIndex) / (N - 1)
    phi_deg = ph0_deg + ph1_deg * g
    phi_rad = phi_deg * pi / 180
    S_corr(n) = S(n) * exp(1i * phi_rad)
end

Output: S_corr
```
